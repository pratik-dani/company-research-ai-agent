# Input Parameters

The Company Research & Analysis Agent accepts a simple input schema with a single required parameter.

## Input Schema

```json
{
  "title": "Company Research Agent Input",
  "type": "object",
  "schemaVersion": 1,
  "properties": {
    "domain": {
      "title": "Company Domain",
      "type": "string",
      "description": "Domain name of the company to research (e.g., apple.com)",
      "editor": "textfield"
    }
  },
  "required": ["domain"]
}
```

## Parameters

<ParamField path="domain" type="string" required>
  The domain name of the company you want to research. This should be the root domain without any protocol or path.
  
  **Examples:**
  - `apple.com`
  - `microsoft.com`
  - `tesla.com`
</ParamField>

## Input Validation

The actor performs the following validation on the input:
1. Checks if the domain is provided
2. Removes protocol (http/https) if present
3. Removes 'www.' prefix if present
4. Removes any trailing path or query parameters
5. Validates the domain format using the Python `validators` library

## Example Input

### Basic Input
```json
{
  "domain": "apple.com"
}
```

### Input with Extra Parts (automatically cleaned)
```json
{
  "domain": "https://www.microsoft.com/en-us"
}
// Will be processed as: "microsoft.com"
```

## Error Cases

The actor will fail with an error message if:
- No domain is provided
- The domain format is invalid
- The domain cannot be resolved

<Note>
The actor automatically cleans and normalizes the input domain, so you don't need to worry about the exact format. However, the domain must be valid and resolvable.
</Note>
